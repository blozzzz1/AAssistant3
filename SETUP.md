# AI Assistant - Setup Guide

## 📋 Требования

- Node.js 16+ и npm
- Аккаунт Supabase
- API ключ Intelligence.io
- API ключ AITunnel (для генерации видео)

## 🚀 Установка

### 1. Клонирование и установка зависимостей

```bash
# Установите зависимости
npm install
```

### 2. Настройка Supabase

1. Создайте проект на [supabase.com](https://supabase.com)
2. **Отключите подтверждение email** (важно!):
   - Перейдите в **Authentication** → **Providers** → **Email**
   - Найдите опцию **"Confirm email"** и отключите её
   - Нажмите **Save**
3. Перейдите в SQL Editor в вашем проекте
4. Скопируйте содержимое файла `supabase-setup.sql` и выполните его
5. **(Опционально)** Для функции удаления аккаунта: скопируйте и выполните `supabase-delete-user-function.sql`
6. Получите URL и Anon Key в Settings → API

### 3. Настройка бэкенда

**Важно:** Все операции с базой данных теперь выполняются через отдельный бэкенд для безопасности.

1. Перейдите в папку `backend`:
```bash
cd backend
```

2. Установите зависимости:
```bash
npm install
```

3. Создайте файл `.env` в папке `backend`:
```env
SUPABASE_URL=your-supabase-project-url
SUPABASE_SERVICE_ROLE_KEY=your-service-role-key
PORT=3001
NODE_ENV=development
FRONTEND_URL=http://localhost:5173
```

**Важно:** Service Role Key можно найти в Supabase Dashboard → Settings → API → service_role key. Этот ключ имеет полный доступ к БД и должен храниться только на бэкенде!

4. Запустите бэкенд:
```bash
npm run dev
```

Подробнее см. `BACKEND_SETUP.md`

### 4. Настройка переменных окружения фронтенда

Создайте файл `.env` в корне проекта:

```env
VITE_SUPABASE_URL=your-supabase-project-url
VITE_SUPABASE_ANON_KEY=your-supabase-anon-key
VITE_AI_API_KEY=your-intelligence-io-api-key
VITE_AITUNNEL_API_KEY=your-aitunnel-api-key
VITE_API_BASE_URL=http://localhost:3001
```

Замените значения на:
- Данные из Supabase (URL и Anon Key)
- API ключ Intelligence.io (будет использоваться всеми пользователями)
- API ключ AITunnel (для моделей Sora 2, Sora 2 Pro, Wan 2.5, будет использоваться всеми пользователями)
- URL бэкенд сервера (по умолчанию http://localhost:3001)

### 5. Запуск приложения

**Важно:** Сначала запустите бэкенд, затем фронтенд!

1. Запустите бэкенд (в папке `backend`):
```bash
cd backend
npm run dev
```

2. Запустите фронтенд (в корне проекта):
```bash
# Режим разработки
npm run dev

# Сборка для продакшена
npm run build

# Предпросмотр продакшен сборки
npm run preview
```

## 📝 Структура базы данных

### Таблица: `chat_sessions`

| Поле           | Тип       | Описание                              |
|----------------|-----------|---------------------------------------|
| id             | UUID      | Уникальный идентификатор сессии       |
| user_id        | UUID      | ID пользователя (FK на auth.users)    |
| title          | TEXT      | Название чата                         |
| selected_model | TEXT      | ID выбранной AI модели                |
| messages       | JSONB     | Массив сообщений чата                 |
| created_at     | TIMESTAMP | Дата создания                         |
| updated_at     | TIMESTAMP | Дата последнего обновления            |

### Row Level Security (RLS)

Таблица `chat_sessions` защищена политиками RLS, которые гарантируют, что:
- Пользователи видят только свои чаты
- Пользователи могут создавать чаты только для себя
- Пользователи могут изменять только свои чаты
- Пользователи могут удалять только свои чаты

## 🔐 Аутентификация

Приложение использует Supabase Auth для:
- Регистрации новых пользователей
- Входа в систему
- Управления сессиями
- Защиты роутов

### Процесс регистрации

1. Пользователь переходит на `/register`
2. Вводит email и пароль
3. Supabase создает учетную запись
4. После регистрации пользователь автоматически входит в систему

### Процесс входа

1. Пользователь переходит на `/login`
2. Вводит email и пароль
3. После успешного входа перенаправляется на главную страницу

## 🔧 Основные возможности

### ✅ Реализовано

- ✅ Регистрация и вход пользователей
- ✅ Защищенные роуты
- ✅ Сохранение чатов в Supabase
- ✅ Сохранение выбранной модели с чатом
- ✅ Голосовой ввод
- ✅ Поддержка изображений (для моделей с vision)
- ✅ Несколько AI моделей на выбор
- ✅ История чатов
- ✅ Адаптивный дизайн
- ✅ Генерация видео из текста (AITunnel)
- ✅ Генерация изображений из текста (AITunnel - GPT Image, DALL-E)

### 🎯 Особенности

1. **Автоматическая синхронизация**: Чаты автоматически сохраняются в базу данных при каждом изменении
2. **Fallback на localStorage**: Если пользователь не авторизован, данные сохраняются локально
3. **Защита данных**: Row Level Security гарантирует, что пользователи видят только свои данные
4. **Выбор модели**: Каждый чат сохраняет выбранную модель, можно менять модель в процессе

## 🐛 Устранение неполадок

### Ошибка подключения к Supabase

Убедитесь, что:
1. Файл `.env` создан и содержит правильные значения
2. URL и ключ скопированы полностью
3. Проект Supabase активен

### Ошибки при регистрации

Supabase требует:
- Минимум 6 символов в пароле
- Валидный email адрес

### "Email not confirmed" при входе

Если вы видите эту ошибку:
1. Перейдите в Supabase Dashboard → Authentication → Providers → Email
2. Отключите опцию "Confirm email"
3. Сохраните изменения
4. Попробуйте войти снова

Либо:
- Проверьте вашу почту на письмо с подтверждением от Supabase
- Перейдите по ссылке в письме для подтверждения email

### Чаты не сохраняются

Проверьте:
1. Бэкенд сервер запущен и доступен
2. SQL скрипт выполнен в Supabase
3. Политики RLS настроены правильно
4. Пользователь авторизован
5. Переменная `VITE_API_BASE_URL` настроена правильно

### Ошибки подключения к бэкенду

Убедитесь, что:
1. Бэкенд сервер запущен на порту 3001 (или другом, указанном в настройках)
2. `VITE_API_BASE_URL` в `.env` фронтенда соответствует URL бэкенда
3. CORS настроен правильно в бэкенде (переменная `FRONTEND_URL`)
